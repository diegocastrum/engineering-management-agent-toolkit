# One-on-Ones

Source: Camille Fournier, *The Manager's Path*, chapters 1 and 4.

1-1s are the operating system of management. Skip them and you stop seeing problems until they're irreversible. The job here is to help the user run 1-1s that actually surface what matters, not status meetings that waste both sides' time.

## When this skill is loaded, do this first

Figure out which mode the user is in:

- **Prep mode** — "I have a 1-1 with X tomorrow, help me prep." → Pull together an agenda.
- **Reflect mode** — "Here's what we discussed, what should I follow up on?" → Capture notes, extract action items, flag patterns.
- **Diagnostic mode** — "My 1-1s feel like status reports" / "X keeps cancelling" / "I don't know what to talk about." → Diagnose the issue against the patterns below.
- **New-relationship mode** — "I just got a new direct report" or "X is transferring to my team." → Use the new-relationship checklist.

If it's not obvious which mode, ask once. Then proceed without making the user re-explain.

## The five 1-1 styles (and when each fits)

A 1-1 is not one thing. Match the style to what's actually needed:

| Style | When it fits | Watch out for |
|---|---|---|
| **To-do list** | Both sides have concrete items to cover; running shared doc works well | Becoming so transactional that the human side disappears |
| **Catch-up** | Direct report drives; open agenda; good for senior ICs | Drifting into therapy or pure complaining |
| **Feedback meeting** | Quarterly cadence on career, growth, behaviors | Treating it as the *only* time feedback happens — it isn't |
| **Progress report** | Mostly for managing managers, or someone on a side project you don't see | If this is your default for ICs, you're wasting both your times |
| **Getting to know you** | New reports, after big life events, when relationship feels thin | Forcing it on people who don't want it |

Most healthy 1-1s mix two or three of these. If every 1-1 is the same style, that's a signal.

## Prep mode

When the user is preparing for a 1-1, gather (or ask for) just enough context, then produce an agenda:

1. **Recency check.** What's happened with this person since the last 1-1? Recent work shipped or stuck? Recent feedback given or owed? Anything from skip-levels, peer comments, code reviews, incidents?
2. **Standing items.** Anything in the running notes doc that's been hanging? Open commitments from either side?
3. **Their agenda.** Most 1-1s should be 50%+ driven by the report. If you don't know what they want to discuss, the first agenda item is "what's on your mind."
4. **Your agenda.** What feedback do you owe them (positive or corrective)? What context do they need from you? What career or growth conversation is overdue?

Output a short agenda — 3–5 items, ordered by what they'd want to discuss first, then what you need to bring up. Flag any item that's been deferred from a previous 1-1 because Fournier's "no surprises" rule means avoidable surprises in reviews come from feedback that should have been given in a 1-1 and wasn't.

## Reflect mode

After a 1-1, help the user:

1. **Capture the takeaways.** What was discussed, what was decided, what's open.
2. **Extract action items** — separately for the user and for the report. Be explicit about owner and rough timeline.
3. **Flag patterns across multiple 1-1s** if the user shares context: is this person bringing the same problem repeatedly? Have you praised them at all this month? Is feedback you intended to give still undelivered?

If the user maintains running notes, write the update in a format they can paste in. If not, suggest starting one — Fournier specifically recommends a shared doc with the manager as note-taker.

## Diagnostic mode

Match the complaint against these patterns from the book:

- **"1-1s feel like status reports"** → Move status to async (email, chat, standup). Ask the report to bring topics. If they have none, ask open prompts: what's frustrating, what's energizing, what would they change about how the team works.
- **"My report has nothing to say"** → Some people genuinely don't need weekly. Ask them. Drop to bi-weekly if mutual. But check first: are they hiding things? Are they disengaged? A senior IC in a groove is different from a junior person who doesn't know what 1-1s are for.
- **"They keep cancelling / I keep cancelling"** → Marc Hedlund's line: "1-1s are like oil changes — skip them and plan to get stranded." Re-anchor the time, mid-morning Tuesday–Thursday is most reliable. Mondays and Fridays get killed by long weekends. Mornings beat afternoons.
- **"Conversations are just complaining"** → Empathetic listening is good. Wallowing isn't. Acknowledge the feeling, then redirect: "what would help here?" or "what's in your control?"
- **"I never know what to bring up"** → This usually means you're not paying enough attention day-to-day. Look at their tickets, code reviews, recent meetings. There's always something to discuss; if there isn't, the relationship is too distant.
- **"They never push back / never disagree"** → You may have a culture-of-fear problem. Test: when's the last time they told you you were wrong? If you can't remember, you have the problem, not them.

## New-relationship mode

When a new person starts reporting in (new hire, transfer, former peer), run a first 1-1 with these questions. Fournier draws several of these from Lara Hogan's blog post; include them all unless the user objects:

- How do you like to be praised — in public or in private?
- What's your preferred medium for serious feedback — written so you can digest it, or verbal in conversation?
- Why did you decide to work here? What are you excited about?
- How will I know when you're in a bad mood or annoyed? Any consistent triggers I should know?
- Are there manager behaviors you know you hate?
- Do you have clear career goals I should know about?
- Any surprises since you joined — good or bad?

Also set up:
- A 30/60/90-day plan with concrete milestones (especially for new hires — surfaces mishires fast).
- Expectations about how the two of you will communicate (frequency, format, what goes in 1-1 vs async).
- An invitation for *their* feedback on you in the first 90 days, while their eyes are still fresh.

**Former peer transitioning to report**: name the awkwardness directly. "This is going to be a weird transition and I'm going to make mistakes — I need you to tell me when I do." Resist the urge to micromanage; they'll be hypersensitive to any "I got rewarded over you" signaling.

## Forcing functions (apply silently)

When generating any 1-1 output, the skill should:

- Bias toward shorter agendas with deeper items over long lists of shallow ones.
- If the user hasn't praised this person in the last 2–3 1-1s (per their notes), surface "what's something they did well recently?" as a prompt.
- If corrective feedback has been deferred more than once, flag it explicitly: this is now the kind of thing that becomes a review-day surprise.
- Treat "everything's fine" reports with mild suspicion — they're sometimes true, but they're also the most common signal of a report who's about to quit (Fournier's "warning signs" pattern).

## What this skill should NOT do

- Don't psychoanalyze the report from secondhand description. Stick to what the user observed.
- Don't generate generic "rapport-building questions" lists. Use the new-relationship checklist or nothing.
- Don't produce HR-style scripts for normal conversations. 1-1s are between two adults; over-scripting kills them.
